Fundamentos de Segurança Web

Aprenda a identificar e explorar vulnerabilidades em aplicações web

Sobre Este Módulo

Este é o primeiro módulo da nossa jornada em segurança cibernética. Aqui você aprenderá os fundamentos das vulnerabilidades mais comuns em aplicações web, como elas funcionam e como podem ser exploradas por atacantes maliciosos.


Duração: Estude no seu ritmo
Nível: Iniciante
Modalidade: Gratuito / Educacional

🔴 SQL Injection (SQLi)

SQL Injection é uma técnica de injeção de código que explora vulnerabilidades em aplicações que usam entrada do usuário em instruções SQL. O atacante pode manipular consultas ao banco de dados para acessar, modificar ou excluir informações não autorizadas.


Tipos de SQL Injection:

  • In-Band SQLi: O ataque e os resultados são retornados na mesma comunicação
  • Blind SQLi: O atacante não vê a resposta direta, mas observa comportamento
  • Out-of-Band SQLi: Usa canais alternativos para obter resultados

Exemplo Clássico:

// Vulnerável:
SELECT * FROM users WHERE username = '" + userInput + "' AND password = '" + passInput + "'

// Ataque (userInput):
admin'--

// Query executada:
SELECT * FROM users WHERE username = 'admin'--' AND password = ''

Como Prevenir:

  • Use Prepared Statements (Parameterized Queries)
  • Valide e sanitize todas as entradas do usuário
  • Use ORM (Object-Relational Mapping)
  • Aplique o princípio do menor privilégio

🟡 Cross-Site Scripting (XSS)

XSS permite que atacantes injetem scripts maliciosos em páginas web visualizadas por outros usuários. Isso pode roubar cookies, sessões, redirecionar usuários ou realizar ações em nome da vítima.


Tipos de XSS:

  • Reflected XSS: Script malicioso é refletido no servidor e executado imediatamente
  • Stored XSS: Script malicioso é armazenado no servidor e executado quando outros usuários acessam
  • DOM-based XSS: Script é executado no lado do cliente, manipulando o DOM

Exemplo de Payload:

// Redirecionamento malicioso:


// Keylogger:

Como Prevenir:

  • Encode todas as saídas (HTML, URL, JavaScript)
  • Use Content Security Policy (CSP)
  • Valide entrada do usuário
  • Use frameworks que fazem escape automático

🟠 Cross-Site Request Forgery (CSRF)

CSRF força usuários autenticados a executar ações não desejadas em aplicações web. O atacante induz o usuário a enviar requisições indesejadas para sites onde está autenticado.


Como Funciona:

  • Usuário faz login no banco online
  • Usuário visita site malicioso (em outra aba)
  • Site malicioso envia requisição automática ao banco
  • Browser do usuário envia cookies de autenticação junto
  • Transferência não autorizada é realizada

Como Prevenir:

  • Use tokens CSRF (Anti-CSRF tokens)
  • Verifique o Referer header
  • SameSite cookies
  • Requisições POST para ações sensíveis

🔵 Broken Authentication

Vulnerabilidades em sistemas de autenticação permitem que atacantes comprometam senhas, chaves de sessão ou explorem falhas de implementação para acessar contas de outros usuários.


Falhas Comuns:

  • Credential Stuffing: Uso de senhas vazadas em outros serviços
  • Session Hijacking: Roubo de ID de sessão
  • Weak Password Policy: Senhas fracas permitidas
  • Session Fixation: Forçar ID de sessão conhecido

Boas Práticas:

  • Implemente MFA (Autenticação Multifator)
  • Use senhas hashadas com salt (bcrypt, argon2)
  • Timeouts de sessão adequados
  • Rate limiting em tentativas de login

🟣 Sensitive Data Exposure

Exposição de dados sensíveis ocorre quando aplicações não protegem adequadamente informações como senhas, dados pessoais, números de cartão, informações financeiras ou dados de saúde.


Cenários de Risco:

  • Dados transmitidos sem criptografia (HTTP em vez de HTTPS)
  • Backups ou arquivos sensíveis expostos publicamente
  • Chaves de API ou segredos hardcoded
  • Banco de dados com dados não criptografados

Proteção:

  • Use TLS/SSL para todas as comunicações
  • Criptografe dados sensíveis em repouso
  • Não armazene dados desnecessários
  • Use secrets management (Vault, AWS Secrets Manager)

🔴 XML External Entity (XXE)

XXE permite que atacantes explorem parsers XML vulneráveis para acessar arquivos locais, realizar SSRF, ou até executar código remoto.

🟡 Security Misconfiguration

Configurações de segurança inadequadas são uma das causas mais comuns de comprometimento. Incluem configurações padrão, recursos desabilitados, ou falta de patches de segurança.


Exemplos de Falhas:

  • Credenciais padrão não alteradas (admin/admin)
  • Debug mode habilitado em produção
  • Arquivos sensíveis expostos (/admin, /.git)
  • CORS mal configurado
  • Headers de segurança ausentes

Checklist de Segurança:

  • Alterar todas as senhas padrão
  • Desabilitar verbose errors em produção
  • Configurar headers de segurança (HSTS, CSP, X-Frame-Options)
  • Remover informações de versão
  • Revisar permissões de arquivos

🟠 Insecure Deserialization

Desserialização insegura ocorre quando dados não confiáveis são desserializados. Atacantes podem manipular objetos serializados para executar código arbitrário.


Impactos:

  • Execução de código remoto (RCE)
  • Injeção de comandos do sistema
  • Acesso a arquivos locais
  • Denial of Service (DoS)

Prevenção:

  • Não aceite objetos serializados de fontes não confiáveis
  • Use formatos de dados seguros (JSON, Protocol Buffers)
  • Valide完整性 dos dados antes de desserializar
  • Execute desserialização em ambientes isolados

🟢 Using Components with Known Vulnerabilities

Componentes de software (bibliotecas, frameworks, plugins) com vulnerabilidades conhecidas são um vetor de ataque comum. Atacantes scaneiam versões vulneráveis ativamente.


Melhores Práticas:

  • Mantenha todos os componentes atualizados
  • Use ferramentas de análise de dependências (OWASP Dependency-Check)
  • Monitore vulnerabilidades conocidas (CVE)
  • Remova componentes não utilizados
  • Prefira bibliotecas com suporte ativo

🔵 Insufficient Logging & Monitoring

Logging e monitoramento insuficientes permitem que ataques passem despercebidos, aumentando o tempo para detecção e resposta a incidentes.


O Que Registrar:

  • Tentativas de login (sucesso e falha)
  • Ações de alto privilégio
  • Erros e exceções
  • Acesso a dados sensíveis
  • Alterações de configuração

Monitoramento Efetivo:

  • Configure alertas para comportamentos suspeitos
  • Integre com SIEM
  • Realize análise de logs regularmente
  • Tenha plano de resposta a incidentes
